## Heartbeat vs Cron 选用指南

| 场景 | 用 | 因 |
|------|-----|-----|
| 批量检查 + 需要上下文 | Heartbeat | 多个检查一次打包，省请求 |
| 精确时间 / 定时任务 | Cron | 精确到秒，可指定不同模型 |
| 倒计时提醒 / 一次触发 | Cron (one-shot) | 用完即走，不占 session |
| 频道独立推送 | Cron (isolated) | 不污染主 session 历史 |

**原则**: 能合到 heartbeat 的就不开单独 cron，减少 token 浪费。
